# rollup迁移webpack4

# 为何要迁移

  • 工具本身属于业务逻辑模块（字体、图片甚至视频）
  • rollup定位
  • rollup本身生态环境的限制
  • webpack强大生态
  • 便于扩展和使用

# 总结

配置项目千变万化，配置套路千篇一律

webpack的默认能力很强大，且生态较为完善，因约定而规范，但也因约定而带来迁移成本。

rollup默认能力一般，工具包构建生态不能应用更多其他场景，但是灵活代码简单。
